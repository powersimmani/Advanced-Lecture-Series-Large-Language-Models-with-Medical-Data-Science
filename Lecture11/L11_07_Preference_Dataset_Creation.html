Preference Dataset Creation
Dataset Structure
Preference datasets contain pairs of model outputs with expert rankings, forming the foundation for reward model training.
Data Components
Input Prompt: Clinical query or task description
Output A: First model-generated response
Output B: Second model-generated response
Preference Label: Which output is preferred (A > B or B > A)
Confidence Score: Strength of preference (optional)
Rationale: Expert's reasoning for choice
Dataset Statistics
Size: 10K-100K+ preference pairs typical
Balance: Equal distribution of A vs B preferences
Coverage: Diverse clinical specialties and scenarios
Quality Metrics: Agreement rates, annotation time
